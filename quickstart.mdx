## Installation

Install wecom-cli globally via your preferred package manager. The postinstall step automatically downloads the correct platform binary.

<CodeGroup>

```bash npm
npm install -g @wecom/cli
```

```bash pnpm
pnpm add -g @wecom/cli
```

```bash yarn
yarn global add @wecom/cli
```

</CodeGroup>

## Setup

<Steps>
  <Step title="Install wecom-cli">
    Run the install command above. After it completes, verify the binary is available:

    ```bash
    wecom-cli --help
    ```
  </Step>
  <Step title="Install AI Skills (optional)">
    If you plan to use wecom-cli with AI agents such as Claude or Cursor, install the bundled skills package:

    ```bash
    npx skills add WeComTeam/wecom-cli -y -g
    ```

    This adds the wecom-cli skills to your global AI tools configuration so agents can call WeCom APIs on your behalf.
  </Step>
  <Step title="Configure your credentials">
    Run the interactive setup wizard. You only need to do this once — credentials are encrypted and stored locally.

    ```bash
    wecom-cli init
    ```

    You will be prompted to choose an authentication method:

    - **QR code (recommended)** — scan the code with the WeCom mobile app to create a bot binding automatically.
    - **Manual** — enter your Bot ID and Secret directly. You can obtain these from the [WeCom developer documentation](https://open.work.weixin.qq.com/help2/pc/cat?doc_id=21677).

    After authentication succeeds, the wizard verifies your credentials against the WeCom API and saves them encrypted to `~/.config/wecom/bot.enc`.
  </Step>
  <Step title="Run your first command">
    Fetch the members visible to your account:

    ```bash
    wecom-cli contact get_userlist '{}'
    ```

    A successful response looks like this:

    ```json
    {
        "errcode": 0,
        "errmsg": "ok",
        "userlist": [
            {
                "userid": "zhangsan",
                "name": "张三",
                "alias": "Sam"
            },
            {
                "userid": "lisi",
                "name": "李四",
                "alias": ""
            }
        ]
    }
    ```

    Each entry contains the member's unique `userid`, display `name`, and optional `alias`.
  </Step>
</Steps>

<Note>
  wecom-cli currently requires a WeCom account in an organization with **10 or fewer members**. If `get_userlist` returns more than 10 members, the API will return an error. Larger organizations are not yet supported.
</Note>

## General command format

Every wecom-cli command follows this pattern:

```bash
wecom-cli <category> <method> [json_args]
```

The supported categories are:

| Category   | Description                              |
| ---------- | ---------------------------------------- |
| `contact`  | Contacts — member queries and search     |
| `doc`      | Documents and smart spreadsheets         |
| `meeting`  | Video meetings                           |
| `msg`      | Messages and media                       |
| `schedule` | Calendar schedules                       |
| `todo`     | Todos and work items                     |

To discover available methods within a category:

```bash
wecom-cli contact --help
```

To see the arguments for a specific method:

```bash
wecom-cli contact get_userlist --help
```
